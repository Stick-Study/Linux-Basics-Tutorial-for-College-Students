Package Management
========
在本章我将主要介绍使用 **A**dvance **P**ackage **T**ool 来进行简单的包管理，主要包括**在线安装**和**安装本地的 `.deb` 文件**。

## APT 的基本用法

### 更新软件源列表

更新软件源列表就好比在 App Store 中**检查更新**，只有在更新软件源列表后才能更新软件本体。使用以下命令来更新软件源列表：
```
sudo apt update
```
该命令会从软件源服务器获取最新的软件列表，但不会**真正安装或更新软件**。

### 升级已安装的软件

在更新软件源列表后，使用以下命令来升级已安装的软件：
```
sudo apt upgrade
```

### 安装软件

使用以下命令安装软件：
```
sudo apt install 软件名
```
例如，安装 `vim` 编辑器：
```
sudo apt install vim
```
安装过程中，系统会提示需要下载的内容和占用的磁盘空间，输入 y 并回车即可继续。

### 卸载软件

卸载软件但保留配置文件：
```
sudo apt remove 软件名
```
例如，卸载 `vim` 编辑器：
```
sudo apt remove vim
```
如果希望连同**配置文件**一起删除：
```
sudo apt purge 软件名
```

### 安装本地 `.deb` 文件

当下载例如微信的 `.deb` 文件后，可以使用 **APT** 安装此文件。操作如下：

首先进入存放 `.deb` 文件的目录:
```
cd 路径
```
然后使用：
```
sudo apt install ./文件名.deb
```
其中 `./` 表示在当前目录下，不可缺少。

## 常用辅助命令

### 搜索软件

当不确定软件的准确名称是，可以使用：
```
apt search 关键字
```
该命令会列出名称或描述中包含关键字的软件包。

### 查看软件信息

查看某个软件包的详细信息：
```
apt show 软件名
```
可以看到版本号、功能描述、依赖关系等信息。

### 查找命令对应的软件包
当你知道一个命令，但不知道它属于哪个软件包时，可以使用：
```
which 命令
```
如果命令存在，会显示其所在路径。

# 更新的 Snap 包管理器
Snap 是 Canonical 推出的一种软件打包与分发方式，Snap 安装的软件自带依赖，不太依赖系统本身的库。这使其软件版本更新快，安装更简单并且依赖问题更少。

## Snap 的基本用法

### 安装 Snap 软件

类似的，使用以下命令：
```
sudo snap install 软件名
```

### 查看已安装的 Snap 软件
```
snap list
```

### 卸载 Snap 软件
```
sudo snap remove 软件名
```

### 更新 Snap 软件
```
sudo snap refresh
```
